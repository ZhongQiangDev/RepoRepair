The issue described is related to the zIndex of the selected feature layer in the `ol.select.Interaction` always appearing on top, regardless of the zIndex set for other layers. This behavior is due to the way the `Select` interaction creates an internal `VectorLayer` for the selected features, which is added to the map and rendered on top of other layers.

To address this, we need to modify the `Select` interaction to allow setting the zIndex of the `featureOverlay_` layer. This can be done by adding a `zIndex` option to the `Select` constructor and setting the `zIndex` property of the `featureOverlay_` layer accordingly.

With these changes, you can now set the `zIndex` option when creating a `Select` interaction to control the rendering order of the selected features layer relative to other layers. For example:

```javascript
const selectInteraction = new ol.interaction.Select({
  // other options...
  zIndex: -1 // Set the zIndex to -1 to render the selected features layer below other layers
});
```
