### 1. Identify the Wrong Behavior
The wrong behavior described in the GitHub issue is that when a `Dropdown` component is set to `disabled`, its child components still have a `tabIndex` of `0`, which allows them to be focused using the tab key. This disrupts the expected flow of tab navigation.

### 2. Analyze the Cause
The cause of this issue is likely due to the `Dropdown` component not properly propagating the `disabled` state to its child components. Specifically, the child components are not having their `tabIndex` set to `-1` when the parent `Dropdown` is disabled, which should prevent them from being focusable.
