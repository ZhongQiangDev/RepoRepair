## Issue Analysis

The issue states that when there are 0 passing audits, the report should not show a details dropdown. Looking at the screenshot, we can see that there's a "View 0 passed items" dropdown, which doesn't make sense since there are no passed items to view.

In the code, the problem is in the `_renderCategory` method. Currently, it always creates a "passed audits" details element with a summary that says "View X passed items", even when there are 0 passed items.

## Solution

We need to modify the `_renderCategory` method to only add the passed audits section if there are actually passed audits to show.
