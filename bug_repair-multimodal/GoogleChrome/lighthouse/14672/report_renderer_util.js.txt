## Bug Analysis

The issue is that DevTools incorrectly shows "no emulation" for mobile runs. According to the issue description, this happened after a change in `getEmulationDescriptions` to return "no emulation" when `screenEmulation.disabled` is true, which is set for DevTools in `devtools-entry.js`.

Looking at the code in `report/renderer/util.js`, the problem is in the `getEmulationDescriptions` method. Currently, it's checking only `screenEmulation.disabled` to determine if it should return "no emulation", but it should also consider the `formFactor` setting to properly identify mobile emulation even when screen emulation is disabled.
