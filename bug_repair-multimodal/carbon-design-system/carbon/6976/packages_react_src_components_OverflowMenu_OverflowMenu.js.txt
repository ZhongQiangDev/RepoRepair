## Issue Analysis

The issue has two parts:
1. `focusable="false"` is being used on a `span` element, but `span` doesn't support the `focusable` attribute
2. `tabindex="0"` is being set on a button element, which is redundant since buttons are already focusable and in the tab order by default

Looking at the code, I can see:
1. In the `iconProps` object, there's a `focusable: 'false'` property being set for the SVG icon
2. In the button element, `tabIndex={tabIndex}` is being used, where `tabIndex` has a default value of `0` in the component's defaultProps

## Explanation of Changes

1. Removed the default `tabIndex: 0` from defaultProps since it's redundant for buttons
2. Replaced `focusable: 'false'` with `aria-hidden: 'true'` for the SVG icon, which is the proper accessibility attribute for hiding decorative elements from screen readers
3. Modified the button element to only include the tabIndex attribute if it's explicitly set to a non-default value (not 0)

These changes address both accessibility issues while maintaining the component's functionality.
