## Issue Analysis

The issue is about accessibility violations in skeleton components (Checkbox, NumberInput, Select, Slider, TextArea & TextInput) detected by DAP (Digital Accessibility Platform). Specifically, the components are missing proper labels.

Looking at the Select.Skeleton.js file, I can see that the `id` prop is being passed to the component and used in the `htmlFor` attribute of the label, but it's not defined in the PropTypes. Additionally, the `id` prop is not being passed to the select element itself, which means the label is not properly associated with the form control.
