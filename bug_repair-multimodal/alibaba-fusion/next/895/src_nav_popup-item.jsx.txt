## Issue Analysis

The issue states that in `iconOnly` mode, menu items without an icon should not appear as empty spaces. Looking at the screenshot, we can see that when no icon is provided, there's still a placeholder space being rendered.

The problem is in the `popup-item.jsx` file. When in `iconOnly` mode, if no icon is provided, the code creates an empty placeholder span:

```javascript
if (iconOnly) {
    if (hasArrow) {
        iconEl = (
            <Icon
                className={`${prefix}nav-icon-only-arrow`}
                type="arrow-right"
            />
        );
    } else if (!iconEl) {
        iconEl = <span className={`${prefix}nav-icon-placeholder`} />;
    }
}
```

This placeholder is causing the empty space to appear. In `iconOnly` mode, items without icons should not be displayed at all.
