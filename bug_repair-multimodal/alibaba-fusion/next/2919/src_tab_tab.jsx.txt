## Issue Analysis

The issue is about the Tab component not having scrolling functionality when the tab content is too long. According to the screenshot and description, the user wants to be able to scroll through tabs using either drag (mouse) or keyboard scrolling when there are many tabs.

Looking at the code, I can see that the Tab component has keyboard navigation for switching between tabs (using arrow keys), but it doesn't have proper scrolling behavior for the tab navigation bar itself when there are many tabs.

The issue is that while the component has a `scrollable` class applied when on touch devices (`isTouchable`), it doesn't enable scrolling for desktop users with many tabs. We need to modify the code to make the tabs scrollable regardless of the device type.
