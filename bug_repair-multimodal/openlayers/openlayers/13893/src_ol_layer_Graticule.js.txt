## Issue Analysis

The issue is about the graticule (grid lines) in OpenLayers showing irregular intervals in the labels. Looking at the screenshot, we can see that the latitude labels show values like 3', 3'.01'', 2'.59'' instead of consistent 3' intervals.

The problem is in how the graticule labels are formatted. The current implementation uses `degreesToStringHDMS` which formats coordinates in degrees, minutes, seconds, and hemisphere. However, when the interval is a non-integer number of degrees (like 3 minutes), the conversion to HDMS format and back can introduce small rounding errors, resulting in the irregular intervals seen in the labels.

## Solution

We need to modify the `lonLabelFormatter_` and `latLabelFormatter_` default functions to ensure they display consistent intervals. The issue is in the `constructor` method of the `Graticule` class where these formatters are defined.

This change ensures that:
1. Very small values close to zero are rounded to exactly zero
2. The formatter is applied consistently to all values
3. The hemisphere designation ('EW' for longitude, 'NS' for latitude) is preserved

The fix addresses the issue by preventing tiny floating-point errors from appearing in the formatted labels, which was causing the irregular intervals in the displayed grid labels.
