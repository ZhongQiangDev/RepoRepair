Based on the issue description and the GIF, the problem occurs when drawing a polygon in OpenLayers 5.3.0. After completing the polygon drawing (on drawend), the map can't be dragged anymore. This issue doesn't happen in OpenLayers 4.6.5.

Looking at the code, I notice that in the `handleUpEvent` method, when a drawing is finished, the `shouldHandle_` flag is set to false, but the `downPx_` property is not reset. This could cause issues with subsequent drag operations.

The issue is that after completing a polygon drawing, the `downPx_` property remains set, which interferes with the normal map dragging behavior. By resetting `downPx_` to null in both the `handleUpEvent` and `finishDrawing` methods, we ensure that after drawing is complete, the map can be properly dragged again.
