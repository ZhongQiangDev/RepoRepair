Based on the issue description, the problem is that the active/inactive state of buttons in the sprite info panel is only indicated by color, which makes it difficult for colorblind users to distinguish between them. The solution proposed is to change the background color of the selected option rather than just coloring the icon.

Looking at the code, I can see that the visibility toggle buttons (show/hide) are implemented in the `SpriteInfo` component. Currently, the active state is only indicated by the `isActive` class, but there's no visual difference in the background that would be perceivable to colorblind users.

This change adds inline styles to set the background color of the active button to blue (#4C97FF) and the inactive button to white. This creates a clear visual distinction between active and inactive states that doesn't rely solely on color saturation, making it more accessible for colorblind users. The blue background for the selected option matches the mockup provided in the issue description.
