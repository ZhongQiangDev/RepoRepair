## Bug Analysis

The issue is with the legend's `onLeave` event not being triggered consistently when the mouse quickly moves outside the chart area. This causes the legend item to remain highlighted even when the cursor is no longer over it.

Looking at the code, the problem is in the `handleEvent` method of the `Legend` class. The current implementation only triggers the `onLeave` event when the mouse moves from one legend item to another, but not when it moves from a legend item to outside the chart area completely.

When the mouse moves quickly outside the chart, the last event received might be a `mousemove` over a legend item, but no subsequent event is received to trigger the `onLeave` callback.

## Fix

We need to modify the `handleEvent` method to properly handle the case when the mouse moves from a legend item to outside the legend area (when `hoveredItem` becomes null).
