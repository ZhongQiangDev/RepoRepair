## Issue Analysis

The issue is about incorrectly rendering labels for vertical pools in BPMN diagrams. When a pool is marked as vertical (`isHorizontal="false"`), the labels are still positioned as if the pool was horizontal, causing incorrect rendering.

Looking at the code in `LabelUtil.js`, I can see that the `getExternalLabelMid` function doesn't account for the orientation of pools when calculating the label position. For vertical pools, we need to adjust the label positioning logic.

## Fix

We need to modify the `getExternalLabelMid` function to check if a pool/lane is vertical and adjust the label position accordingly.

    return {
      x: element.x + element.width / 2,
