## Issue Analysis

The issue is about correctly rendering pool/lane labels in vertical collaboration diagrams. When a pool is marked as vertical (`isHorizontal="false"`), the labels are not properly positioned. The problem is in the `ElementFactory.js` file, which doesn't account for the orientation of pools when determining their default size.

When a pool is vertical, its width and height should be swapped compared to a horizontal pool. Currently, the code doesn't check the orientation of the pool when determining its default size.

## Solution

We need to modify the `getDefaultSize` method in `ElementFactory.js` to check if a participant (pool) or lane is horizontal or not, and swap the width and height accordingly.
