### 1. Identify the Wrong Behavior Involved in `GitHub Problem Description`

The wrong behavior described in the GitHub problem is that when multiple Boundary Events are selected and moved out of their host shape in a batch operation, they are not converted to Intermediate Events as expected. Instead, they remain as Boundary Events, which is incorrect according to BPMN standards.

### 2. Analyze the Cause Behind the Wrong Behavior

The cause of this issue likely lies in the logic that handles the movement of elements within the BPMN diagram. Specifically, the code responsible for detecting when a Boundary Event is moved out of its host shape and converting it to an Intermediate Event is not functioning correctly during batch moves. This could be due to a lack of proper event handling or a missing condition that checks if the moved elements are Boundary Events and if they have been detached from their host.
